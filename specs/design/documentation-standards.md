# 文档规范与工作流

> 本文档定义 Sibylla 项目的文档撰写规范、Spec 工作流和 Changelog 维护规则。

---

## 一、文档命名规范

### 1.1 文件命名
- 全小写，单词间用短横线连接
- 示例：`writing-prd.md`、`analysis-competitor.md`
- 系统文件以下划线开头：`_spec.md`、`_index.md`

### 1.2 文件夹命名
- 全小写，单词间用短横线连接
- 示例：`docs/product/`、`personal/alice/`

---

## 二、Markdown 撰写规范

### 2.1 标题层级
- H1：文档标题（每个文件仅一个）
- H2：主要章节
- H3：子章节
- H4：细分内容

### 2.2 格式约定
- 列表项之间不空行
- 代码块标注语言类型
- 表格对齐使用管道符
- 图片使用相对路径引用 `assets/` 目录
- 文件内链接使用 `[[文件名]]` 语法

---

## 三、Spec 工作流

### 3.1 Spec 流程（适用于需要完整规划的功能）

```
需求提出 → 需求梳理(requirements.md) → 技术方案(design.md) → 任务拆分(tasks.md) → 执行追踪
```

### 3.2 核心文档模板

#### requirements.md 模板
```markdown
# 需求文档

## 介绍
简要描述需求背景和目标。

## 需求

### 需求 1 - 需求名称
**用户故事：** 作为[角色]，我想要[功能]，以便[价值]

#### 验收标准
1. While [前置条件], when [触发器], the [系统] shall [响应]
```

#### design.md 模板
```markdown
# 技术方案设计

## 架构设计
架构图和说明

## 技术选型
选型说明和理由

## 数据模型
数据库/文件结构设计

## 接口设计
API 接口定义
```

#### tasks.md 模板
```markdown
# 实施计划

- [ ] 1. 任务标题
  - 具体要做的事情
  - _需求: 需求 1_
  - _负责人: Alice_
```

## 四、Changelog 维护规则

### 4.1 格式
```markdown
# 变更日志

## 2024-01-15

### 新增
- 完成会员体系 PRD 初稿 (@Alice)

### 变更
- 更新积分权重配置 (@Bob)
```

### 4.2 维护规则
- AI 监测文件变更，自动生成变更条目
- 仅记录有实质意义的变更
- 每条记录标注操作者
- 按日期倒序排列

---

## 五、Spec 文件层级

| 层级 | 文件 | 加载时机 |
|------|------|---------|
| Workspace 级 | `CLAUDE.md` | AI 始终加载 |
| 部门/组级 | `docs/product/_spec.md` | 处理该文件夹内文件时加载 |
| 个人级 | `personal/alice/_spec.md` | 该成员对话时加载 |
