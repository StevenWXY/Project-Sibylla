# UI/UX 设计规范

> 本文档定义 Sibylla 的界面布局、视觉规范和交互设计原则。
> 所有 UI 设计必须遵循 CLAUDE.md 中的 UI/UX 红线。

---

## 一、设计原则

1. **非技术用户优先**：无指导下可完成核心流程（创建 workspace → 导入文件 → AI 对话）
2. **操作可见性**：超过 2 秒的操作必须有进度反馈
3. **安全写入**：文件修改必须展示 diff 预览，禁止静默写入
4. **Git 不可见**：界面中不出现 branch/merge/commit 等术语

### 2.2 区域定义

| 区域 | 宽度 | 说明 |
|------|------|------|
| 左栏 | 220px，可折叠 | 文件树、搜索、任务、通知入口 |
| 中栏 | 自适应 | 编辑区 / AI 对话窗，支持多 Tab |
| 右栏 | 320px，可折叠 | AI 侧边栏 / 评论列表 |
| 顶栏 | 48px | Workspace 切换、同步状态、用户头像 |
| 底栏 | 32px | AI 模式、模型选择、积分、同步状态 |

## 五、核心交互规范

### 5.1 文件状态标识

| 状态 | 图标 | 颜色 | 说明 |
|------|------|------|------|
| 已同步 | ✓ | Emerald | 文件与云端一致 |
| 同步中 | ↻ | Blue | 正在推送/拉取 |
| 本地修改 | ● | Amber | 有未同步的修改 |
| 有冲突 | ⚠ | Red | 需要解决冲突 |
| 待审核 | ◎ | Indigo | 已提交等待审批 |

### 5.2 AI 交互规范

- AI 回复使用流式输出，逐字显示
- AI 建议修改文件时，必须展示 diff 预览（绿色增加、红色删除）
- 用户可选择"应用"或"编辑后应用"
- AI 加载中显示骨架屏 + 脉冲动画
- AI 错误时显示友好提示 + 重试按钮

## 六、组件规范

### 6.1 按钮

- 主按钮：品牌色填充，白色文字，圆角 6px
- 次按钮：白色填充，品牌色边框和文字
- 危险按钮：红色填充，用于删除等不可逆操作
- 幽灵按钮：无边框无背景，仅文字

### 6.2 输入框

- 高度 36px，圆角 6px
- 聚焦时品牌色边框 + 浅色阴影
- 错误态红色边框 + 错误提示文字

### 6.3 Toast 通知

- 右上角弹出，3 秒自动消失
- 成功：绿色左边框
- 错误：红色左边框
- 信息：蓝色左边框

### 6.4 模态框

- 居中显示，背景遮罩 50% 透明度
- 最大宽度 560px
- 危险操作需二次确认
